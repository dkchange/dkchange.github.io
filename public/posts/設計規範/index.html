設計規範也是為了可讀性和可維護性 PSR 是 PHP Standard Recommendations 的簡寫，由 PHP FIG 組織制定的 PHP 規範，是 PHP 開發的實踐標準。

小到命名規則，大到namespace的應用 https://www.php-fig.org/personnel/ 在上面的網址可以看到一些有名的專案和專案的貢獻者，例如 guzzle

程序员中的单个方法的行数极限是几行？

這個問題本質是在於可讀性，和維護性，不要誇張即可 与具体语言与业务逻辑相关的，C写一个标准的双层switch状态机实现，80行以内就是不可能的。大函数实现状态机核心逻辑非常常见。 但是面向对象的语言可能会要求你用多态类而非双重switch方式实现状态机，这样就可以控制在80行内。
