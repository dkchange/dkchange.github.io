前言

http://vito-note.blogspot.com/2012/01/blog-post_86.html http://www.ruanyifeng.com/blog/2007/10/ascii_unicode_and_utf-8.html 平常在 coding 的我們，最常接觸的東西，就是打字啦，每一種編碼標準都是為了轉換 byte value 與 character，使用不同的編碼方式，同一个二进制数字可以被解释成不同的符號。 每一種特定的語言，都存在一種或數種的字集碼，每一種字集碼就是一個代碼頁 ( code page )。例如：big5 的 code page = 950

Unicode

但卻無法在同一個文件裏面同時支援多個語系。 Unicode 是為了解決傳統的字元編碼方案的侷限所產生的一項業界標準 它只规定了符号的二进制代码，却没有规定这个二进制代码应该如何存储。

16 進制

因為用二進制來對應所有的字實在太長 汉字严的 Unicode 是十六进制数 4E25 转换成二进制数足足有 15 位（100111000100101） 所以 unicode 採用 16 進制來做對應 Unicode 当然是一个很大的集合，现在的规模可以容纳 100 多万个符号。 你覺得字元要用多少 bit 才夠，如何有辦法對應所有的語言的字母 大概 2 的 21 次方就可以容納 1999999 個字，也就是 2bytes 到 3bytes 之間 \u4e2d\u6587 這是中文兩個字鎖對應的 unicode

UTF-8

UTF-8 是 Unicode 的实现方式之一，最大的一个特点，就是它是一种变长的编码方式。它可以使用 1~4 个字节表示一个符号 \xE4\xB8\xAD\xE6\x96\x87 這是中文兩個字鎖對應的 UTF-8

UTF-16

UTF-16 是 Unicode 的实现方式之一，字符用两个字节或四个字节表示 \u{4e2d}\u{6587} 這是中文兩個字鎖對應的 UTF-16

UTF-32

UTF-32 是 Unicode 的实现方式之一，字符用四个字节表示

那由以上來看，utf-8 要筆其他兩種方式都省空間 U+00004e2dU+00006587 這是中文兩個字鎖對應的 UTF-16

16 進位的轉換

假設你有一個 string str=“12345” 你想要加一個字元 enter 他的 16 進位編碼是 0A “x31\x32\x33\x34\x35\x0A"貼在瀏覽器會變成

"x312345
"

查看

$ od -xc “filename” | head

a ：利用預設的字元來輸出； c ：使用 ASCII 字元來輸出 d[size] ：利用十進位(decimal)來輸出資料，每個整數佔用 size bytes ； f[size] ：利用浮點數值(floating)來輸出資料，每個數佔用 size bytes ； o[size] ：利用八進位(octal)來輸出資料，每個整數佔用 size bytes ； x[size] ：利用十六進位(hexadecimal)來輸出資料，每個整數佔用 size bytes ；
